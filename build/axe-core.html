Axe Manifesto

Automated accessibility testing must

  1. Have zero false positives

  2. Be lightweight and fast

  3. Work in all modern browsers

  4. Be tested automatically itself

Versatility of the axe

  • Plugins for browsers

Embedded in HTML

<script>
  window.onload = function () {
    axe.a11yCheck(document, {
      runOnly: { type: "tag", values: [
        "wcag2a", "wcag2aa", "best-practice"
      ]}
    }, (result) => console.log(result));
  }
</script>

What aXe tests

The 54 rules to date

  • Text alternatives and accessible names

  • Validity of your ARIA

  • Correct structure (tables, lists, title)

  • Zoom restrictions, auto-refresh

  • Color contrast, link visibility

  • ... and more

But not everything

  • Dynamic content

  • Descriptiveness of text

  • Error and timeout handling

  • Inappropriate semantics

Test coverage
